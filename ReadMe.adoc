# KALYPSO – Smart Archaeological Shelter System  
### From Data to Decision. From Concept to Reality.

---

## 🧠 Project Overview

KALYPSO is a smart, autonomous archaeological shelter system designed to improve the safety, efficiency, and data collection process during excavations.

In the FIRST LEGO League UNEARTHED™ challenge, we identified that archaeologists:
- Work in harsh environmental conditions (heat, humidity, rain)
- Lack integrated systems for real-time data collection
- Still rely on manual documentation processes

KALYPSO transforms a simple excavation tent into an **intelligent, connected IoT system** that:
- Protects archaeologists
- Collects environmental and excavation data
- Supports real-time decision making

---

## ⚙️ System Architecture

KALYPSO follows a **Sense → Analyze → Decide → Act** pipeline:

### 🔹 Sense
- Light sensors (LDR)
- Temperature & Humidity (DHT20)
- Rain sensor
- IMU (movement/orientation)
- GPS module
- AI Camera (HuskyLens)

### 🔹 Analyze
- Arduino GIGA WiFi processes data
- AI camera detects findings
- Environmental monitoring logic

### 🔹 Decide
- Autonomous decisions based on:
  - Sun direction
  - Weather conditions
  - Detected findings

### 🔹 Act
- Servo motors adjust solar panel
- Curtain system activates for protection
- Cooling fan system
- Notifications sent via IoT Cloud

---

## 🔌 Hardware Components

- **Microcontroller:** Arduino GIGA WiFi  
- **AI Vision:** HuskyLens AI Camera  
- **Sensors:**
  - LDR (light tracking)
  - DHT20 (temperature & humidity)
  - Raindrop sensor
  - IMU sensor
  - GPS module  
- **Actuators:**
  - Servo motors (solar tracking)
  - DC motors (curtain system)
  - Cooling fan  
- **Display:** LCD screen  
- **Connectivity:** WiFi + IoT Cloud

== 🔌 Wiring & Connections (Simplified Schematic)

The system is built around the Arduino GIGA WiFi, which connects to all sensors, actuators, and modules.

=== Sensors

[cols="1,1,1", options="header"]
|===
| Component | Connection Type | Arduino Pins

| LDR Sensors | Analog Input | A0, A1
| DHT20 | I2C | SDA / SCL
| Rain Sensor | Digital Input | D2
| IMU Sensor | I2C | SDA / SCL
| GPS Module | UART | RX / TX
| HuskyLens (AI) | I2C / UART | SDA / SCL or Serial
|===

=== Actuators

[cols="1,1,1", options="header"]
|===
| Component | Connection Type | Arduino Pins

| Servo Motors | PWM | D5, D6
| Curtain Motor | Digital Output | D7, D8
| Cooling Fan | Digital Output | D9
|===

=== Display

[cols="1,1", options="header"]
|===
| Component | Connection

| LCD | I2C (SDA / SCL)
|===

## 💻 Software Structure

The system is modular and organized into multiple `.ino` files:

### 🔹 `sensors.ino`
Handles all environmental data:
- Reads temperature, humidity, light, rain
- Collects GPS and IMU data

### 🔹 `ai.ino`
- Controls HuskyLens AI camera
- Detects archaeological findings
- Processes object recognition data

### 🔹 `actuators.ino`
- Controls motors and servos
- Activates curtain system
- Manages cooling and movement

### 🔹 `servo.ino`
- Handles solar panel tracking
- Adjusts angle based on light sensors

### 🔹 `cloud.ino`
- IoT communication
- Sends real-time data to cloud
- Handles notifications (push alerts)

### 🔹 `lcd.ino`
- Displays system status
- Shows environmental data

### 🔹 `helpers.ino`
- Utility functions
- Shared logic across modules

### 🔹 `simulation.ino` / `simulation.h`
- Testing environment
- Simulates system behavior

### 🔹 `thingProperties.h`
- IoT Cloud configuration
